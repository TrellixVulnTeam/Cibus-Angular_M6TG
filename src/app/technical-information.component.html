How it began


Cibus started out as a class project. The objective was to build a production-ready, responsive web application using full-stack technologies in order to help consumers create healthy meals inspired by environmental sustainable strategies and small business ecosystems. I initially worked on this project with two other classmates. Together, we interviewed the client, collected and analyzed business requirements, translated them into functional requirements, and created technical documentation â€“ a UML Use Case diagram, a UML Component diagram, an Entity-Relationship diagram, and wireframes. Finally, we developed an MVP using a multi-layered architecture consisting of a presentation tier, a Node.js middle-tier API, and a data tier that leverages a SQLite relational database for persistence.




What it has become


Once the course ended, I continued working on this project alone. After multiple iterations, Cibus evolved into a streamlined experience where end users can instantly look up existing recipes, create new ones, search for ingredients, and get in touch with local providers. A blog was also added to share information about food trends and cooking tips, and to provide a space for the exchange of ideas, opinions, and experiences. This required modifying the inital database design to accomodate additional tables as well as updating the API accordingly to process new requests such as retrieving articles, posting comments, etc.
